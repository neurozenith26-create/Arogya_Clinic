Request lifecycle (e.g. patient opens /doctors)

  1. Browser hits https://arogya-clinic.netlify.app/doctors → Netlify serves index.html + the React bundle.
  2. React app boots, React Query calls GET https://arogya-clinic.onrender.com/api/v1/doctors using VITE_API_URL baked into the bundle at build time.
  3. Render's Express app validates Origin against CORS_ORIGIN and replies with Access-Control-Allow-Origin: https://arogya-clinic.netlify.app.
  4. Express runs SELECT … FROM users WHERE role='doctor' AND is_active AND is_verified on the pg pool.
  5. The pool talks to Supabase's session pooler (TLS) → Supabase returns rows.
  6. Express returns { data: [...] } as JSON. React Query caches; component renders.
Files (report PDFs, payment proofs, profile photos): stored as BYTEA inside Postgres tables (migrations 0022, 0024, 0027). No object storage; Render's disk is ephemeral and intentionally unused.

# 3. Commit & push — backend code that uses
