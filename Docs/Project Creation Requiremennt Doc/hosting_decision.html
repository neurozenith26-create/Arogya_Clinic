1. Why not use Netlify's own DB?

Netlify DB is just Neon Postgres rebranded. It's a plain database with no auth, no file storage, no realtime, no RLS dashboard. Your codebase is already deeply wired into Supabase, which gives you all of that for free.

Feature your app usesSupabaseNetlify DB (Neon)
Postgres database✓ Yes✓ Yes
Built-in Auth (SUPABASE_ANON_KEY)✓ Yes✗ Build your own
Service role for admin operations✓ Yes✗ Not available
File / image storage (lab reports)✓ Supabase Storage✗ Need separate S3
Row Level Security via dashboard✓ Yes✗ Manual SQL only
Realtime subscriptions✓ Yes✗ No
@supabase/supabase-js client✓ Works as-is✗ Code rewrite

Switching cost

  • Rewrite the entire auth layer (currently free with Supabase)
  • Replace frontend/src/lib/supabaseClient.ts and every import of it
  • Migrate file uploads to S3/Cloudflare R2
  • Re-run migrations against Neon
  • Lose RLS dashboard, lose realtime, lose Storage

Benefit of switching: essentially zero. Both are Postgres on AWS. Netlify DB is Neon with a thinner feature set.

2. Why not use Netlify Functions as backend?

Netlify Functions are short-lived serverless functions (AWS Lambda). Your Express backend is a long-running stateful server. They don't fit together without major rewrites.

What your backend usesProblem on Netlify Functions
node-cron for scheduled tasksFunctions can't run on schedule — need to rewrite as Netlify Scheduled Functions
express-rate-limit with in-memory storeIn-memory counters reset every call — rate limiting silently breaks
Long-lived pg connection poolPool resets on every cold start → connection storms hit Supabase
multer file uploadsHard 6 MB request body limit
pdfkit PDF generation10s timeout on free tier — risky for big PDFs
Razorpay webhook handlerCold starts can cause webhook timeouts
Graceful SIGTERM shutdownIrrelevant — functions just die
Two route modules at /api/v1Must restructure as one-function-per-route, or use serverless-http wrapper

What it would cost you

  • Multi-day refactor of the backend
  • Add external Redis (Upstash) for rate limiting → extra service to pay for
  • Restructure folder layout to netlify/functions/
  • Handle 6 MB upload ceiling via presigned uploads
  • End up with a less capable backend than you have today

Verdict: Render runs your Express app as-is. No rewrites, no compromises.

3. Advantages of each service

🌐 Netlify (Frontend)

  • Global CDN — fast page loads worldwide
  • Auto-deploy on git push — every commit deploys
  • Free HTTPS — automatic TLS certificates
  • Deploy previews — every PR gets its own URL
  • Instant rollbacks — one-click to any previous deploy
  • Custom domains free — bring your own domain
  • SPA redirects — React Router works with one config line
  • 100 GB bandwidth / month free — ~100,000 page views

🚀 Render (Backend)

  • Runs your Express app unmodified — no serverless rewrites
  • Persistent server — node-cron, websockets, anything works
  • Auto-deploy from GitHub
  • Free HTTPS + custom domains
  • Built-in health checks — uses your /api/v1/health
  • Zero-downtime deploys — old instance stays up till new one passes health check
  • Easy env var management
  • Logs + metrics in dashboard

🗄️ Supabase (Database)

  • Managed Postgres — backups, patching handled for you
  • Built-in Auth — email, OAuth, magic links, OTP
  • File Storage — for lab reports, scans, profile images
  • Row Level Security UI — secure data access without writing middleware
  • Realtime subscriptions — push DB changes to clients
  • Auto-generated REST + GraphQL APIs
  • SQL editor in dashboard
  • 500 MB DB + 1 GB storage free

5. Summary

✓ Cheapest production-grade setup ✓ Zero code rewrites required ✓ Each service does what it's best at ✓ Auto-deploys from GitHub ✓ Scales when you do

Use Netlify for what it's great at: shipping static frontends fast.
Use Render for what it's great at: running Node.js servers.
Use Supabase for what it's great at: being a full backend-as-a-service for data, auth, and files.

Don't force any one of them to do the others' job — that's where the rewrites, costs, and broken features pile up.
